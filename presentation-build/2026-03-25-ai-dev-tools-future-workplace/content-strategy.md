# 프레젠테이션 콘텐츠 전략

**주제**: AI 개발 도구 시대의 기술적 도약, 산업 파급력, 미래 일터, 준비 전략
**대상 청중**: 비즈니스/기술 리더, 개발팀 관리자, 기술 기획 담당자
**슬라이드 수**: 28장
**작성일**: 2026-03-25

---

## 1. 핵심 메시지 (5개)

**메시지 1: AI 개발 도구는 "자동완성"에서 "에이전틱 코딩"으로 도약했으나, 생산성 향상은 조건부다**
- 단순 과제 +30~50%, 조직 수준은 프로세스 변화 없이 10~15%가 상한
- So what? → 도구 도입만으로는 부족하다. 프로세스 재설계가 투자 회수의 열쇠다

**메시지 2: 채택의 진짜 병목은 기술이 아니라 조직 거버넌스, 프로세스, 보안 체계다**
- DORA 2025: AI는 기존 조직의 강점과 약점을 증폭시키는 "증폭기"
- So what? → 도구 선택보다 조직 역량 점검이 선행되어야 한다

**메시지 3: 개발자 역할이 코딩에서 오케스트레이션으로 전환되고 있다**
- 주니어 고용 13~20% 감소, 시니어/AI 전문가 비중 증가
- So what? → 인재 전략과 역할 재정의를 지금 시작해야 한다

**메시지 4: AI 코드 보안 부채가 조용히 누적되고 있다**
- AI 생성 코드 취약점 비율 높으나 인간 코드 대비 비교치 부재. 볼륨 증가가 핵심 리스크
- So what? → 생산성 추구와 보안 거버넌스를 동시에 설계해야 한다

**메시지 5: 4가지 시나리오에 대비하되, "아무것도 하지 않는 것"이 가장 위험한 선택이다**
- 기술 정체 확률 10~15%, 나머지 85~90%에서 AI 준비가 경쟁력을 좌우
- So what? → 기준 시나리오(점진적 통합) 중심으로 설계하되, 방어를 내장하라

## 2. 서사 구조 선택 이유

**SCQA 구조 (Situation → Complication → Question → Answer)**를 기반으로 하되, 기술 기획 보고서 성격에 맞게 **미래 전략 요소를 강화**한다.

선택 이유:
- 청중(비즈니스/기술 리더)은 "현황 → 문제 → 질문 → 해답" 흐름에 익숙하다
- 단순 트렌드 보고가 아닌 "우리가 무엇을 해야 하는가"로 귀결되어야 한다
- SCQA 후반부에 시나리오와 로드맵을 배치하여 전략적 깊이를 확보한다

서사 흐름:
1. **Situation** (현재): AI 개발 도구가 어디까지 왔는가 (기술 도약의 실체)
2. **Complication** (문제): 생산성 역설, 보안 부채, 역할 변화라는 세 가지 도전
3. **Question** (질문): 미래 일터는 어떻게 변하는가? 우리는 무엇을 준비해야 하는가?
4. **Answer** (답변): 시나리오별 전략 + 구체적 로드맵 + 즉시 실행 항목

## 3. 파트별 스토리라인

### Part 0: 오프닝 (1장)
- **목적**: 주제 설정, 청중의 관심 포착
- **슬라이드 수**: 1장

### Part 1: Situation — AI 개발 도구의 현재 (5장)
- **목적**: 청중이 AI 개발 도구 생태계의 규모와 기술 도약 수준을 이해하게 한다
- **핵심 전달**: 시장 3강 구도, 자동완성→에이전틱 진화, 채택률 85~95%, 코드의 41~46%가 AI 지원
- **슬라이드 수**: 5장
- **전환 논리**: "이렇게 빠르게 확산되었는데, 정말 생산성이 높아졌을까?"

### Part 2: Complication — 세 가지 역설과 도전 (7장)
- **목적**: 낙관론에 균형을 잡는다. 생산성 역설, 보안 부채, 신뢰 역설을 직면한다
- **핵심 전달**: 조건부 생산성(+30~50% vs -19%), AI 코드 보안 리스크, 조직 수준 정체, 신뢰 하락+사용 증가 역설
- **슬라이드 수**: 7장
- **전환 논리**: "기술이 아닌 조직이 병목이라면, 산업과 일터는 어떻게 변하는가?"

### Part 3: Question — 미래 일터의 구조 변화 (5장)
- **목적**: 팀 축소, 주니어 감소, 역할 전환, 비소프트웨어 산업 파급을 보여준다
- **핵심 전달**: 팀 규모 축소, 주니어→시니어 구성 변화, "빌더" 역할 부상, 산업별 파급
- **슬라이드 수**: 5장
- **전환 논리**: "이 변화에 어떻게 대비해야 하는가?"

### Part 4: Answer — 시나리오와 전략 (8장)
- **목적**: 4가지 시나리오를 제시하고, 시나리오별 전략 옵션과 구체적 로드맵을 제공한다
- **핵심 전달**: 시나리오 매트릭스, 기업 전략(지금/실험/관망/회피), 개인 전략, 로드맵
- **슬라이드 수**: 8장
- **전환 논리**: "구체적으로 무엇을 모니터링하고, 언제 전략을 조정할 것인가?"

### Part 5: 클로징 — 핵심 메시지와 모니터링 (2장)
- **목적**: 핵심 교훈 정리, 선행지표/전환 트리거 제시, 행동 촉구
- **슬라이드 수**: 2장

  - 실험: 프로세스 재설계 + 에이전틱 코딩 제한적 도입 + 역할 재정의
  - 관망: 멀티에이전트 전면 도입, 오픈소스 자체 호스팅
  - 회피: 프로세스 없는 전사 확산, 거버넌스 없는 에이전트, 주니어 대량 해고
- 소스: 92-strategy-options.md:기업 전략 옵션, 00-synthesis.md:섹션 8

**[23] [Content] "프로세스 재설계가 생산성 10~15%를 20~30%로 끌어올리는 열쇠다"**
- Bain: "도구만 도입하면 10~15%가 상한"
- SDLC 전 주기 AI 통합: 요구사항→설계→코딩→테스트→배포
- 성공 사례: Goldman Sachs(사유 코드베이스 파인튜닝), 상위 사분위 기업(65% 일일 사용→15%+ 속도)
- 실패 사례: 27% 프로덕션 코드 AI지만 10% 생산성만 달성
- 소스: 03-preparation-strategy.md:섹션 3.1~3.2, 섹션 4

**[24] [Content] "개인 전략: AI 시대에 대체 불가능한 포지션을 확립하라"**
- 3계층 스킬: (1) AI 리터러시+도구 숙련(즉시), (2) 시스템 사고+아키텍처(12개월), (3) 도메인xAI 융합(장기)
- 즉시: AI 도구 일상화 + 비판적 검토 + 포트폴리오 기록
- 3~12개월: 도메인 x AI 전문성 + 에이전트 조율 역량
- 상시 병행: AI 없는 기초 역량 유지
- 소스: 03-preparation-strategy.md:섹션 2, 92-strategy-options.md:개인 전략 옵션

**[25] [Timeline] "기술 로드맵: 0~3개월 / 3~12개월 / 12~36개월"**
