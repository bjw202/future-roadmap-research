# 콘텐츠 전략 레퍼런스

## 액션 타이틀 원칙

모든 슬라이드 제목은 결론 문장이어야 합니다. 청중이 제목만 읽어도 핵심 메시지를 이해할 수 있어야 합니다.

### 변환 예시

| ❌ 주제형 (Bad) | ✅ 액션형 (Good) |
| --- | --- |
| "매출 분석" | "Q3 매출이 전년 대비 23% 성장했습니다" |
| "경쟁사 비교" | "품질과 서비스에서 경쟁사 대비 우위를 확보했습니다" |
| "시장 현황" | "국내 시장 규모가 2025년 1조원을 돌파했습니다" |
| "팀 소개" | "10명의 전문가 팀이 프로젝트를 이끕니다" |
| "문제점" | "현재 프로세스에서 3가지 병목이 생산성을 저하시킵니다" |
| "해결 방안" | "자동화 도입으로 처리 시간을 60% 단축할 수 있습니다" |

### 액션 타이틀 작성 공식

1. 핵심 데이터/사실을 먼저 파악
2. "무엇이 어떻게 되었는가" 형태로 작성
3. 가능하면 수치 포함 (신뢰도 향상)
4. 한국어 기준 30자 이내를 목표로 하되, 핵심 메시지 전달이 우선. addTitleBar가 autoFit으로 긴 제목도 자동 처리한다

## 프레젠테이션 구조 패턴

### 1. SCQA 비즈니스 보고

```
표지
├─ 상황(Situation): 현재 상태 파악
├─ 문제(Complication): 해결해야 할 도전
├─ 질문(Question): 핵심 질문 제시
├─ 답변(Answer): 권고사항/해결책
├─ 근거: 데이터, 분석, 증거
└─ 결론 및 다음 단계
```

### 2. 피치덱 구조

```
표지
├─ 문제: 고객이 겪는 고통 (Pain)
├─ 해결책: 우리의 접근방식
├─ 시장 규모: TAM/SAM/SOM
├─ 비즈니스 모델: 수익 창출 방법
├─ 견인력: 현재까지의 성과/지표
├─ 경쟁 분석: 차별화 포인트
├─ 팀: 핵심 인물 소개
├─ 재무 계획: 3~5년 예측
└─ 클로징: 요청사항 (CTA)
```

### 3. 교육/강의 구조

```
표지
├─ 학습 목표
├─ 목차
├─ [섹션 1] 개념 이해
│   ├─ 핵심 개념 설명
│   ├─ 예시/실습
│   └─ 핵심 요약
├─ [섹션 2~N] (반복)
└─ 전체 요약 및 Q&A
```

### 4. 학술/연구 발표

```
표지
├─ 연구 배경 및 필요성
├─ 연구 목적 및 가설
├─ 연구 방법론
├─ 연구 결과 (데이터)
├─ 결과 해석 및 토론
├─ 결론 및 함의
└─ 참고문헌
```

## 서사 비트 시스템 연동

위 4가지 구조는 서사 비트 시스템의 구체적 구현이다. 각 구조를 비트 시퀀스로 매핑하면 감정 흐름을 체계적으로 설계할 수 있다. 상세 비트 정의와 청중별 매핑은 `references/narrative-beats.md` 참조.

| 기존 구조 | 비트 시퀀스 |
| --- | --- |
| SCQA | hook → problem → insight → proof → close |
| 피치덱 | hook → problem → insight → comparison → proof → close |
| 교육 | hook → framework(×N, 섹션별 insight 반복) → proof → close |
| 학술 | hook → problem → framework → proof → insight → close |

**사용법:** 청중 유형이 명확하면 `narrative-beats.md`의 청중별 비트 순서를 우선 적용. 목적이 명확하면 위 구조별 비트 매핑을 적용. 아웃라인 작성 시 각 슬라이드에 `[비트]` 주석을 추가하여 감정 흐름을 시각화한다.

## 피라미드 원칙

맥킨지 MECE 원칙 적용:

```
결론 (Conclusion)
├─ 근거 1 (Key Point 1)
│   ├─ 증거 A
│   └─ 증거 B
├─ 근거 2 (Key Point 2)
│   ├─ 증거 C
│   └─ 증거 D
└─ 근거 3 (Key Point 3)
    ├─ 증거 E
    └─ 증거 F
```

- 결론 먼저 제시 (Top-down)
- 각 레벨은 MECE (상호배타, 전체포괄)
- 3의 법칙: 근거는 3개가 이상적

## 콘텐츠 밀도 제어

### 6x6 규칙

- 슬라이드당 글머리 기호 최대 6개
- 글머리당 단어 6개 이내 (한글 15\~18자)
- 한 슬라이드 = 핵심 메시지 1개

### 슬라이드 분할 기준

텍스트가 다음 조건 충족 시 분할:

- 글머리 기호 7개 이상
- 두 가지 이상의 독립된 메시지
- 스크롤 없이 읽기 어려운 분량

### 데이터 표현 최적화

- 숫자는 단순화: 12,345,678 → 1,234만
- 비율은 시각화: 67.3% → 차트 사용
- 비교는 나란히: 텍스트 나열 → 테이블

## 슬라이드별 콘텐츠 가이드

### Title Slide

- 제목: 발표 주제를 결론 문장으로 (44pt)
- 부제목: 범위/기간/맥락 (20pt)
- 발표자/날짜: 간결하게 (14pt)

### Section Divider

- 섹션 번호: 01, 02, 03... 스타일
- 섹션 제목: 해당 섹션의 핵심 메시지 (36pt)
- 설명: 1\~2문장으로 섹션 내용 예고 (16pt)

### Content (일반 내용)

- 제목: 해당 슬라이드 핵심 결론
- 본문: 글머리 기호 3\~5개, 간결한 키워드/구문
- 시각 보조: 관련 이미지, 아이콘

### KPI Dashboard

- KPI 숫자: 핵심 지표 3\~4개만
- 라벨: 짧고 명확하게 (5자 이내)
- 변화율: +/- 표시로 방향성 전달

### Data Table

- 행 수: 최대 8\~10행 (가독성 한계)
- 열 수: 최대 6\~7열
- 초과 시: 중요도 낮은 열 제거 또는 분할

### Chart + Insight

- 차트: 한 가지 메시지만 전달
- 인사이트: 차트에서 발견한 핵심 3가지
- 인사이트 형식: 수치 강조 + 짧은 설명

### Card Grid

- 카드당 제목: 5\~8자
- 카드당 본문: 2\~3줄
- 카드 수: 2x2 (4개) 또는 2x3 (6개)
- 카드 내용: 서로 독립적이고 동등한 수준

### Timeline

- 항목 수: 4\~7개 (가독성 최적)
- 날짜/단계: 명확하게 표시
- 각 항목: 제목 + 짧은 설명 (1\~2줄)

### Quote Slide

- 인용문: 조선일보명조 22pt, italic
- 출처: 우측 하단, Pretendard 14pt
- 배경: 미니멀 (단색 또는 연한 그라데이션)

### Closing

- 핵심 메시지 3가지 요약
- 다음 단계 (Next Steps) 명시
- CTA: 명확한 행동 촉구 문구
- 연락처/QR 코드 (해당되는 경우)
