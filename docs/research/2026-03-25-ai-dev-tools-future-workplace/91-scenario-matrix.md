## 시나리오 A: 기준 시나리오 — "점진적 통합"

> AI 기술이 현재 궤적대로 발전하고, 조직은 시행착오를 거치며 점진적으로 수용

| 항목 | 내용 |
|------|------|
| **전제** | AI 모델 성능 연 15~25% 개선 지속, 조직 거버넌스가 1~2년 내 정립, 규제는 점진적 강화 |
| **핵심 변화** | 팀 규모 20~30% 축소가 3년 내 주류화. 에이전틱 코딩이 반복 업무의 40~50% 대체. 주니어 개발자 역할이 "AI 감독 하 실행"으로 재정의. 비개발자 프로토타이핑 확산 |
| **수혜 분배** | 대기업 > 플랫폼 > 시니어 개발자 > 주니어 개발자 |
| **생산성 실현** | 프로세스 변화 병행 시 20~30%, 미병행 시 10~15% |
| **촉발 조건** | SWE-bench 실세계 버전 60%+ 달성, 주요 기업 AI 거버넌스 프레임워크 공개 |
| **반증 조건** | 2027년까지 대형 AI 코드 보안 사고 발생으로 신뢰 후퇴 |
| **확률 판단** | [중간~높음] 50~60% |

## 시나리오 B: 낙관 시나리오 — "소프트웨어 수요 폭발"

> AI 도구가 진입 장벽을 낮춰 모든 산업의 소프트웨어화를 가속, 전체 수요 폭증

| 항목 | 내용 |
|------|------|
| **전제** | AI 성능이 급격히 향상(SWE-bench 90%+), 규제가 혁신 친화적, 비용이 현재의 1/10 이하로 하락 |
| **핵심 변화** | 모든 산업에서 AI 기반 자체 소프트웨어 구축 가속. 개발자 총 수요 오히려 증가. "빌더" 직무 보편화. AI 에이전트가 전체 SDLC의 70%+ 자동 처리 |
| **수혜 분배** | 플랫폼 > 얼리어답터 기업 > 시민 개발자 > 전통 개발자 |
| **생산성 실현** | 40~60% (새로운 카테고리의 소프트웨어 탄생으로 측정 기준 자체 변화) |
| **촉발 조건** | AI 노코드 도구가 복잡 시스템 구축 가능 수준 도달, 추론 비용 현재의 1/100, 주요국 AI 규제 샌드박스 확산 |
| **반증 조건** | AI 모델 성능 포화(scaling law 한계), 에너지/인프라 제약으로 비용 반등 |
| **확률 판단** | [낮음~중간] 15~25% |

---

## 시나리오 C: 보수 시나리오 — "규제 브레이크"

> 대형 보안 사고 또는 법적 분쟁으로 규제가 급격히 강화, 채택 속도 둔화

| 항목 | 내용 |
|------|------|
| **전제** | AI 생성 코드 기반 대형 보안 사고 발생, EU AI Act 엄격 집행, 미국도 AI 코드 책임 법안 통과 |
| **핵심 변화** | AI 코드 인간 검토 의무화. 컴플라이언스 비용 급증. AI 도구 채택 속도 50% 둔화. 거버넌스/컴플라이언스 전문 직무 대규모 창출. 산업별 격차 심화(규제 산업은 채택 동결, 비규제 산업은 지속) |
| **수혜 분배** | 컴플라이언스 도구 기업 > 시니어 개발자(검토 역할 강화) > 대기업(자체 검증 인프라) |
| **생산성 실현** | 5~10% (컴플라이언스 오버헤드로 이득 상쇄) |
| **촉발 조건** | Fortune 500 기업에서 AI 생성 코드 취약점 기반 대형 침해 사고, 또는 AI 저작권 소송 패소 |
| **반증 조건** | 주요국이 AI 규제 완화를 경쟁력 정책으로 전환, AI 보안 도구가 취약점을 인간 수준 이하로 감소 |
| **확률 판단** | [낮음~중간] 15~20% |

## 시나리오 D: 기술 정체 시나리오 — "고원기"

> AI 모델 성능이 현재 수준(SWE-bench 80%)에서 포화, 에이전틱 코딩이 프로덕션 신뢰성을 달성하지 못함

| 항목 | 내용 |
|------|------|
| **전제** | Scaling law 한계 도달. SWE-bench 80%에서 실질적 포화. 실세계 복잡 코드베이스에서 성공률 30~40% 수준 정체. 에너지/인프라 비용이 하락 멈춤 |
| **핵심 변화** | AI 도구가 고급 자동완성~대화형 코딩 수준에 머무름. 에이전틱/멀티에이전트는 실험 수준. 개발자 역할 변화 완만. 주니어 채용 감소 추세 둔화. 기업의 AI 도구 구독 해지 증가 |
| **수혜 분배** | 실용적 도구 채택 기업 > 인간 중심 프로세스 유지 기업 > 과잉 투자 회피 기업 |
| **생산성 실현** | 10~20% (현재 확인된 수준에서 고착) |
| **촉발 조건** | 2027년까지 SWE-bench 85% 미돌파, 에이전트 프로덕션 성공 사례 소수 |
| **반증 조건** | 새로운 아키텍처(test-time compute, memory augmentation 등)로 성능 급등 |
| **확률 판단** | [낮음] 10~15% |

## 공통 모니터링 지표

시나리오에 관계없이 추적해야 하는 선행지표:

1. **SWE-bench 실세계 버전 점수 추이** — 기술 정체 vs 가속 판단
2. **Fortune 500 AI 코드 보안 사고 발생 여부** — 규제 브레이크 트리거
3. **Gartner Hype Cycle에서 AI 코딩 도구의 위치** — 기대 조정 시점
4. **AI 도구 구독 해지율(churn rate)** — ROI 실현 여부
5. **주니어 개발자 채용 비율 추이** — 구조 변화 속도
6. **EU AI Act 최초 대규모 벌금 시점** — 규제 실효성
7. **오픈소스 모델 vs 상용 모델 벤치마크 격차** — 생태계 구조 변화
