**리서치 주제**: AI 개발 도구 및 AI 기반 업무 도구 생태계의 미래
**작성일**: 2026-03-25
**작성자**: Signal Analyst
**조사 모드**: Strategic Foresight Mode

## 핵심 병목 진단 (Question Expansion §6)

현재 AI 개발 도구 생태계의 핵심 병목은 **기술 자체**가 아니다. 모델 성능은 이미 전문가 수준에 근접했다. 진짜 병목은:

1. **조직 병목**: 거버넌스 부재, Shadow AI, 기술 수용 역량(AI 리터러시)
2. **보안/신뢰 병목**: AI 생성 코드의 품질 검증 체계 미성숙
3. **표준화 병목**: 에이전트 간 프로토콜(MCP, A2A) 안착 여부
4. **데이터 병목**: 기업 내부 데이터와 AI 연결의 컴플라이언스 문제

### 신호 1: MCP·A2A 프로토콜의 사실상 표준화

| 필드 | 내용 |
|------|------|
| **신호** | Anthropic이 개발한 Model Context Protocol(MCP)이 2025년 12월 Linux Foundation AAIF에 기증된 후 OpenAI·Google·Microsoft·AWS가 모두 채택. 2026년 2월 기준 월 9,700만 SDK 다운로드(Python+TypeScript 합산). Google의 Agent-to-Agent(A2A) 프로토콜과 함께 "3계층 AI 프로토콜 스택(MCP→도구, A2A→에이전트 간, WebMCP→웹)"이 사실상 표준으로 수렴 중 |
| **카테고리** | 기술/표준화 |
| **출처** | [The New Stack, "Choosing Your AI Orchestration Stack for 2026", 2026-03], [MCP vs A2A Guide, dev.to, 2026] |
| **강도** | 강 |
| **시점** | 현재 (수렴 진행 중) |
| **불확실성** | 낮 (주요 플레이어 전원 참여 확인) |
| **우리에게 중요한 이유** | 승자 포지션이 "모델"이 아닌 "에이전트 생태계 인프라(MCP 서버, 도구 레이어)"로 이동. 지금 MCP 호환 도구를 보유하지 않으면 후발 탑승 비용 증가. 플랫폼 의존도(Lock-in)가 MCP 생태계로 재편될 위험 |
| **전환 트리거** | MCP가 ISO/IETF 공식 표준으로 등재되거나, 규제 기관이 AI 시스템 간 상호운용성 인증에 MCP를 요구하는 시점 |
| **반증 조건** | A2A 또는 AWS 독자 표준이 엔터프라이즈 채택에서 MCP를 앞서거나, 프로토콜 분열로 인해 사실상 표준이 3개 이상으로 분산되는 경우 |
